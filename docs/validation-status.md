# Validation status

This document records maintained contract status, not every development binary
hash. Exact AkuSupervisor SHA-256, runtime version, timestamps, checks, and local
paths are generated by the Rust runner under `.artifacts/reports` and remain
ignored. Update this file only when a fixture contract or maintained tuple
changes.

## Windows application-owned fixtures

Status: **Node.js, Go, and Rust deterministic and native gates pass**.

All three contract-version-1 fixtures were revalidated on 2026-07-17 through
AkuSupervisor `0.1.0` using AkuSupervisorConformance `0.2.0` report schema `2`.
Each proof independently established:

- the deterministic shutdown test passed;
- the direct application executable became healthy;
- the independent application endpoint became reachable;
- AkuSupervisor delivered targeted Ctrl+Break;
- the application recorded its expected native signal and every declared
  cleanup event;
- shutdown reported `gracefulSignalSent: true`, `forced: false`, and an empty
  `ownedPidsAfter`;
- the lifecycle journal carried matching evidence;
- the listener was released; and
- an unrelated Rust-runner sentinel remained alive.

Runtime-specific signal observations were:

| Fixture | Windows application observation |
|---|---|
| Node.js application-owned | `SIGBREAK` |
| Go application-owned | `SIGINT` / `os.Interrupt` |
| Rust application-owned | `CTRL_BREAK_EVENT` |

The Go fixture deliberately registers both `os.Interrupt` and
`syscall.SIGTERM`. Go maps Control-C and Control-Break to `os.Interrupt`; console
close, logoff, and shutdown notifications use the distinct `SIGTERM` path.

The Rust fixture is independent of AkuSupervisor internals. It owns a minimal
HTTP server, request draining, a background worker, resource evidence, and a
Windows console handler; it does not import AkuSupervisor's
`platform::windows` adapter.

The Node fixture continues to cover the corrected native launch boundary:
AkuSupervisor retains the real primary-thread handle, removes only its own
suspend count after Job assignment, uses `NUL` stdin, and captures bounded
stdout/stderr without resuming antivirus/EDR-owned threads.

These maintained claims are Windows-only. They do not imply Linux or macOS
certification, and Conformance remains separate from AkuSupervisor build and
stable promotion.
